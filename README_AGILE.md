# Agile/Scrum Workflow for Church Management System

## Sprint Planning
- Define sprint goal (1-2 weeks)
- Break down features into user stories/tasks
- Assign tasks to team members

## Daily Standup
- What did you do yesterday?
- What will you do today?
- Any blockers?

## Sprint Review
- Demo completed features
- Get feedback from stakeholders

## Sprint Retrospective
- What went well?
- What can be improved?
- Action items for next sprint

## Sample Board (Trello/Notion/Jira)
- Backlog
- To Do
- In Progress
- Review
- Done

---
You can use [Trello](https://trello.com/), [Notion](https://notion.so/), or [GitHub Projects](https://github.com/features/project-management/) for your board.
